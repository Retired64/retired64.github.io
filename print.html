Si prefieres organizar tus archivos Markdown dentro de una carpeta llamada src junto con el archivo SUMMARY.md, puedes hacerlo para mantener una estructura más ordenada.

Para lograr esto, después de ejecutar mdbook init, puedes crear manualmente la carpeta src y mover tus archivos Markdown allí. Por ejemplo:

  1. Después de ejecutar mdbook init, crea una carpeta llamada src:

    mkdir src

  2. Mueve tus archivos Markdown (por ejemplo, capitulo_1.md, capitulo_2.md, etc.) a la carpeta src:

    mv capitulo_1.md src/
mv capitulo_2.md src/

  3. Asegúrate de que el archivo SUMMARY.md esté en la raíz del directorio del libro. Si se creó dentro de src al principio, muévelo a la raíz:

    mv src/SUMMARY.md ./

Esto organizará tus archivos Markdown dentro de la carpeta src junto con el archivo SUMMARY.md en la raíz del directorio de tu libro. Esta estructura puede ayudar a mantener una mejor organización, especialmente si tienes varios archivos Markdown para tu libro.

¡Claro! Aquí tienes una guía detallada sobre cómo utilizar mdbook en Termux, desde lo básico hasta las funcionalidades más avanzadas:

Instalación de mdbook en Termux:

  1. Abre Termux.
  2. Instala mdbook usando pkg: 
    pkg install mdbook

Creación de un nuevo libro:

  1. Crea un directorio para tu libro:

    mkdir mi_libro
cd mi_libro

  2. Inicializa un nuevo libro usando mdbook:

    mdbook init

    Esto creará la estructura básica de tu libro en el directorio mi_libro.

Edición del contenido del libro:

  1. Utiliza tu editor de texto preferido para editar los archivos Markdown en el directorio de tu libro. Puedes organizarlos en carpetas si lo prefieres.

  2. Asegúrate de que el archivo SUMMARY.md tenga la estructura deseada, enlazando a tus archivos Markdown.

Previsualización local del libro:

  1. Para previsualizar tu libro localmente, ejecuta:

    mdbook serve

    Esto iniciará un servidor local para tu libro en http://localhost:3000.

  2. Abre un navegador web en Termux o en cualquier dispositivo en la misma red y visita esa dirección para ver cómo se ve tu libro.

Generación del libro final:

  1. Cuando hayas terminado tu libro y estés listo para generar la versión final, ejecuta: 
    mdbook build
    Esto generará los archivos HTML estáticos de tu libro en un directorio llamado book.

Visualización del libro generado:

  1. Abre el archivo index.html dentro del directorio book en un navegador web desde Termux para ver el libro finalizado.

Funcionalidades avanzadas:

  • Personaliza el aspecto de tu libro utilizando temas y estilos específicos.
  • Agrega complementos (plugins) a mdbook para extender su funcionalidad (como soporte para diagramas, ecuaciones, etc.).
  • Utiliza Git para el control de versiones de tu libro y despliega tu libro en plataformas como GitHub Pages.

¡Con estos pasos, podrás empezar desde cero y avanzar hacia un uso más experto de mdbook en Termux para crear y compartir tu libro!

Entendido, si deseas ubicar todos los archivos, incluido el SUMMARY.md, dentro de la carpeta src, puedes organizarlos de esta manera:

  1. Estructura de carpetas sugerida: Coloca todos tus archivos, incluyendo SUMMARY.md, dentro de la carpeta src en el directorio principal de tu libro.

La estructura de carpetas debería verse así:

mi_libro/
└── book.toml
└── book/
└── src/
     ├── SUMMARY.md
     ├── introduccion.md
     ├── capitulo_1.md
     └── capitulo_2.md

  1. Ejecución del comando mdbook build:

    • Abre Termux y asegúrate de estar en el directorio raíz de tu libro (mi_libro en este caso).
    • Ejecuta el comando mdbook build en este directorio.
    cd ruta/a/tu/libro/mi_libro
mdbook build

El comando mdbook build generará los archivos HTML estáticos de tu libro en un directorio llamado book dentro del directorio raíz de tu libro (mi_libro). Para acceder al libro generado, abre el archivo index.html dentro del directorio book en un navegador web desde Termux o transfiriendo estos archivos a un servidor web para su visualización en línea.

¡Claro! Puedes utilizar servicios de alojamiento de páginas web gratuitos, como GitHub Pages, para convertir tu carpeta book en un sitio web accesible en línea de forma gratuita.

Aquí te muestro cómo hacerlo utilizando GitHub Pages:

Pasos para desplegar tu libro usando GitHub Pages:

  1. Crear un repositorio en GitHub:

    • Ve a GitHub y crea una cuenta si no tienes una.
    • Crea un nuevo repositorio haciendo clic en el botón "New" en tu perfil.

  2. Subir tu carpeta book al repositorio:

    • Sube la carpeta book, que contiene los archivos HTML estáticos de tu libro, a este repositorio.

  3. Habilitar GitHub Pages:

    • Ve a la pestaña "Settings" (Configuraciones) de tu repositorio.
    • Desplázate hacia abajo hasta encontrar la sección "GitHub Pages".
    • En la opción "Source" (Origen), selecciona la rama que contiene la carpeta book.
    • Guarda los cambios.

  4. Acceder a tu libro en GitHub Pages:

    • Después de unos minutos, GitHub Pages generará automáticamente un enlace en la sección "GitHub Pages" de la configuración de tu repositorio.
    • Puedes acceder a tu libro mediante el enlace proporcionado (usualmente tiene el formato https://tu_nombre_de_usuario.github.io/tu_repositorio).

Una vez completados estos pasos, tu libro generado con mdbook estará disponible como una página web gratuita en GitHub Pages. Los usuarios podrán acceder a él utilizando el enlace generado por GitHub.
